# Review Patterns Reference

Patterns and strategies for reviewing discoveries.

## Review Frameworks

### CARE Framework

**C**larity - Is the message clear?
**A**ccuracy - Is the data correct?
**R**elevance - Does it matter to user?
**E**ffectiveness - Is it actionable?

### Quick Assessment

For routine reviews:

| Check | Pass | Fail |
|-------|------|------|
| Clear headline | ✓ | Vague/confusing |
| Data support | ✓ | No evidence |
| Actionable | ✓ | Just info |
| Timely | ✓ | Stale data |

### Deep Assessment

For complex findings:

1. **Methodology Review**
   - How was this found?
   - Is approach valid?
   - Any biases?

2. **Data Validation**
   - Source reliable?
   - Data complete?
   - Calculations correct?

3. **Conclusion Check**
   - Does data support claim?
   - Alternative explanations?
   - Appropriate confidence?

4. **Impact Assessment**
   - Who is affected?
   - How significant?
   - What action needed?

## Review by Discovery Type

### Insight Reviews

Focus on:
- Pattern validity
- Statistical significance
- Business relevance
- Actionability

Questions:
- Is the trend real?
- Is the change significant?
- So what? What should user do?

### Alert Reviews

Focus on:
- Urgency accuracy
- Threshold appropriateness
- Context completeness
- Response clarity

Questions:
- Is this truly urgent?
- Is the threshold right?
- What should user do now?

### Summary Reviews

Focus on:
- Completeness
- Balance
- Key highlights
- Trends accuracy

Questions:
- Are key points covered?
- Is tone appropriate?
- Are trends correctly characterized?

### FYI Reviews

Focus on:
- Necessity
- Clarity
- Brevity
- Timing

Questions:
- Does user need to know this?
- Is it clear and brief?
- Is timing appropriate?

## Batch Review Strategies

### Prioritize First

Sort discoveries by:
1. Urgency (alerts first)
2. Impact (high impact next)
3. Confidence (higher first)
4. Age (older first)

### Group by Topic

Review related discoveries together:
- Same metric
- Same data source
- Same time period
- Related themes

### Apply Templates

Use consistent criteria:
- Same quality bar
- Same rejection reasons
- Same approval rationale

## Common Review Decisions

### Approve When

- Clear, supported insight
- Actionable for user
- Appropriate timing
- Matches user interests

### Reject When

- Vague or unclear
- No data support
- Not actionable
- Duplicate content
- Wrong timing

### Request Revision When

- Good insight, poor presentation
- Missing key context
- Needs more detail
- Tone adjustment needed

## Review Metrics

### Quality Indicators

| Metric | Target |
|--------|--------|
| Approval rate | 70-85% |
| User useful rate | > 60% |
| Rejection for quality | < 20% |
| Time to review | < 1 hour |

### Warning Signs

- Approval rate > 95% (not filtering enough)
- Approval rate < 50% (generation issues)
- Low useful rate (quality issues)
- Long review times (process issues)
