# Example Reference

This is a template for reference files. Reference files provide detailed information that would make the main SKILL.md too long.

## Guidelines for References

- Keep files focused on a single topic
- Use descriptive filenames (e.g., `filter-operators.md`, not `doc1.md`)
- Add a table of contents for files over 100 lines
- References must be one level deep only (no nested folders)
